## XP 个人主页 — 产品规格文档

> 最后更新: 2026-06-17
> 状态: 已对齐，待实现

---

### 产品定位

将个人主页（linktree 风格）包装为一台虚拟 Windows XP 系统。用户在浏览器中看到的不是一个普通网页，而是一个可交互的 XP 桌面环境——有壁纸、任务栏、开始菜单、可拖拽的程序窗口。所有个人内容（简介、AI 对话、产品 PDF、联系方式）都在这个虚拟系统中呈现。

---

### 桌面环境层（始终存在，不可关闭）

| 元素 | 说明 |
|------|------|
| 像素风 Bliss 壁纸 | canvas 渲染，8px 像素块风格，蓝天+白云+绿色山丘。全屏固定。 |
| XP 任务栏 | 固定在底部，30px 高。包含 Start 按钮、任务按钮区、系统托盘、实时时钟。 |
| 语言切换按钮 | 固定在右上角，XP 按钮风格。切换中/英文。 |
| Toast 提示 | 浮动在最上层，XP 黄色工具提示风格。 |

---

### Start 菜单

点击 Start 按钮弹出。分为三个区域：

**头部** — 用户头像 + 名字（蓝色渐变背景）。

**左栏（程序列表）** — 4 个真实入口：

| 菜单项 | 点击行为 |
|--------|----------|
| System Properties | 打开/聚焦「个人主页」窗口 |
| 海外大模型中转.pdf | 打开/聚焦对应 PDF 窗口 |
| FastGPT 商业版.pdf | 打开/聚焦对应 PDF 窗口 |
| Microsoft 365.pdf | 打开/聚焦对应 PDF 窗口 |

**右栏（系统位置）** — 3 个装饰性窗口入口：

| 菜单项 | 点击行为 |
|--------|----------|
| My Computer | 打开「我的电脑」装饰窗口（像素风模拟界面，含驱动器图标等） |
| My Documents | 打开「我的文档」装饰窗口（像素风模拟文件夹列表） |
| Control Panel | 打开「控制面板」装饰窗口（像素风模拟分类面板） |

**其他项**（Internet Explorer、Outlook Express、Notepad、Search、Help、Run）— 纯装饰，点击无反应。

**底部** — Log Off / Shut Down 按钮（纯装饰）。

### 窗口清单

#### 窗口 1：个人主页（System Properties）

- **触发**: 页面加载自动打开；也可从 Start 菜单打开
- **内容**: 头像 + 姓名 + 标语、AI 对话入口按钮、4 个产品条目、联系方式图标、页脚
- **底部按钮**: OK / Cancel / Apply（装饰性）
- **可关闭**: 是，关闭后可从 Start 菜单重新打开
- **标签页**: General / Computer Name / Hardware / Advanced / 主要项目

**窗口内的操作触发其他窗口**:

| 操作 | 触发窗口 |
|------|----------|
| 点击「AI 对话」按钮 | 打开/聚焦「AI 聊天」窗口 |
| 点击「AI 定制解决方案」条目 | 打开/聚焦「AI 聊天」窗口并自动发送引导语 |
| 点击「海外大模型中转」条目 | 打开/聚焦「海外大模型 PDF」窗口 |
| 点击「FastGPT 商业版」条目 | 打开/聚焦「FastGPT PDF」窗口 |
| 点击「Microsoft 365 Business」条目 | 打开/聚焦「M365 PDF」窗口 |
| 点击联系方式图标 | 复制到剪贴板 + Toast 提示（不打开新窗口） |

#### 窗口 2：AI 聊天

- **触发**: 从主窗口内的按钮触发
- **内容**: 标题栏（蓝色渐变）、消息流区域、推荐问题 chips、输入框 + 发送按钮
- **可关闭**: 是
- **流式对话**: 保留现有的 SSE 流式输出逻辑

#### 窗口 3/4/5：PDF 查看器（×3）

- **触发**: 从主窗口内的产品条目或 Start 菜单触发
- **内容**: 标题栏（显示 PDF 名称）、iframe 嵌入对应 PDF 文档
- **可关闭**: 是
- **iframe 源**: `/pdf/apihub-overseas-llm-relay.pdf`、`/pdf/fastgpt-commercial.pdf`、`/pdf/microsoft365-copilot.pdf`

#### 装饰窗口（×3）：My Computer / My Documents / Control Panel

- **触发**: 仅从 Start 菜单右栏触发
- **内容**: 像素风模拟界面（纯 CSS/SVG 绘制，无真实功能）
  - My Computer: 驱动器图标网格（C:、D:、光驱等）
  - My Documents: 文件夹列表样式
  - Control Panel: 分类面板样式（类别图标 + 名称）
- **可关闭**: 是
- **可拖拽**: 是（与功能窗口行为一致）

### 窗口管理器行为规则

| 行为 | 规则 |
|------|------|
| 拖拽 | 按住标题栏拖动窗口位置，释放停止 |
| 聚焦 | 点击窗口任意位置 → 该窗口提升到最顶层（z-index 最高），标题栏高亮 |
| 最小化 | 点击任务栏中已激活窗口的按钮 → 窗口隐藏，任务栏按钮变为非激活态 |
| 还原 | 点击任务栏中非激活窗口的按钮 → 窗口显示并聚焦 |
| 关闭 | 点击标题栏 ✕ 按钮 → 窗口销毁，任务栏按钮移除 |
| 新窗口位置 | 居中偏右下错开，避免完全重叠 |
| 最大窗口数 | 理论上限 8 个（5 功能 + 3 装饰），实际不太会同时全开 |

### 移动端策略（≤768px）

| 元素 | 行为 |
|------|------|
| 任务栏 | 隐藏 |
| Start 菜单 | 隐藏 |
| 桌面壁纸 | 隐藏（窗口铺满屏幕） |
| 主窗口 | 全屏铺满，无拖拽，无窗口装饰边框圆角 |
| AI 聊天 | 大型对话框覆盖约 85% 屏幕（非全屏覆盖），居中显示，带标题栏和关闭按钮 |
| PDF 条目 | 直接在浏览器新标签页打开 PDF（iframe 在移动端体验差） |
| 语言切换 | 保留，右上角小按钮 |
| Toast | 保留，底部居中 |

---

### 技术约束

- 框架: Astro 4.x 静态构建，所有窗口管理器逻辑为客户端原生 JS
- 数据源: 所有文案和产品数据仍由 `src/config/content.json` 统一管理
- i18n: 中英双语，`data-i18n` 属性绑定，localStorage 持久化语言选择
- AI 对话: 仍通过 `/api/chat` SSE 流式接口，由 Node 代理转发 FastGPT
- PDF 嵌入: 桌面端用 `<iframe>`，移动端降级为 `window.open()`
- 无新依赖: 窗口管理器纯 JS 实现，不引入 React/Vue/第三方 UI 库
